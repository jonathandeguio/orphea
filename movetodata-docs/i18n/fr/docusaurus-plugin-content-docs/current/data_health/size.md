# La taille

#### Objectif

Les contrôles de la taille garantissent que la taille de l'ensemble de données se situe dans la plage attendue. Ils aident à détecter les anomalies telles que les données manquantes ou les ajouts inattendus.

#### Configuration

Pour configurer un contrôle de la taille, vous devez définir la plage de taille attendue des données. Voici un exemple de configuration :

**Exemple de contrôle de la taille**

- **Nom :** Validation de la taille de l'ensemble de données
- **Type :** Taille
- **Critères :**
  - Taille minimale : 10 Mo
  - Taille maximale : 100 Mo

#### Surveillance

Les contrôles de la taille peuvent être surveillés via le tableau de bord de l'application. Des alertes peuvent être configurées pour informer les parties prenantes lorsque la taille de l'ensemble de données se situe en dehors de la plage attendue.

#### Bonnes pratiques

- **Définir des plages claires :** Spécifiez des limites de taille minimales et maximales claires pour chaque ensemble de données.
- **Surveillance régulière :** Surveillez en continu les tailles des ensembles de données pour identifier rapidement les anomalies.
- **Ajuster les critères :** Examinez et ajustez régulièrement les critères de taille pour les aligner avec les volumes de données changeants.
