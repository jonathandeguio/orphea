#  Compliations

<div style="text-align: justify"> Une compliations est le processus de compilation du code source dans une application logicielle ou une bibliothèque exécutable.

Le processus de génération implique généralement de compiler le code, de le lier à des bibliothèques externes et de créer un package installable pouvant être déployé dans un environnement cible. Le processus de construction peut être automatisé à l'aide d'outils de construction comme Gradle. L'automatisation du processus de construction ainsi que la planification peuvent faire gagner du temps et assurer la cohérence entre les environnements.

Vous pouvez démarrer le processus de construction après avoir mis à jour un jeu de données particulier.

<b>YVous cliquez simplement sur Build qui est l'icône du marteau ici et le processus commence automatiquement.</b>

![Builds](../../_media/Transform/transform/builds%20fr.png)


<b><i><u>Remarque :</i></u> Il s'agit de la version normale.</b>

### <b>Journal de construction</b>

<p style="text-indent: 20px;">Vous pouvez également afficher le journal de construction qui fournit un enregistrement du processus de construction utilisé pour suivre la progression de la construction du logiciel. Un journal de génération inclut généralement des informations sur le processus de génération, telles que les commandes exécutées, les résultats de chaque commande et les erreurs ou avertissements qui se sont produits pendant la génération.</p>

![BuildLog](../../_media/Transform/builds/buildlog%20fr.png)

<p style="text-indent: 20px;">Les journaux de compilation sont utiles pour un certain nombre de raisons. Ils fournissent un enregistrement détaillé du processus de génération, qui peut être utilisé à des fins de débogage si des erreurs se produisent. En analysant les journaux de build, les développeurs peuvent identifier les tendances et les modèles dans le processus de build, ce qui peut aider à identifier les opportunités d'amélioration.</p>
